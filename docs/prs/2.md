# PR: Issue 2 - Retarget toolchain to Textual

Branch: `2-retarget-toolchain-textual`. Source: `docs/plans/ascent.md`.

## Summary

Move the presentation stack from pygame to Textual and establish the core/ui
layering seam the rest of the slice builds on. The runtime dependency becomes
`textual` in place of `pygame-ce`, the pygame headless test configuration is
removed, and `ascent.core` becomes a real (initially empty) package guarded by a
test asserting it never imports `textual`. There is no game behaviour in this
issue; it retargets the scaffold from issue 1 to the new stack.

## How it was tested

The behavioural surface is one test: the core/ui layering guard. The remaining
acceptance criteria are toolchain gates, verified by the full `pytest` suite,
`ruff check`, `ruff format --check`, and `mypy` all exiting 0 on a clean `.venv`,
and by `python3 -m ascent --version` still working. Those gate results are
recorded in the GREEN cycle once the retarget is applied.

## Cycle 1: RED

**Test surface**

- `tests/core/test_layering.py::test_importing_core_does_not_load_textual` runs
  `import ascent.core` in a fresh subprocess and asserts `textual` is absent from
  `sys.modules`. The subprocess isolates the check so an unrelated import in the
  session cannot mask a real layering violation.

**Coverage**

| Acceptance criterion | Covered by |
|---|---|
| Importing `ascent.core` leaves `textual` out of `sys.modules` | `test_layering.py` (RED now, GREEN after `ascent.core` exists) |
| `pyproject.toml` depends on `textual`, not `pygame-ce` | install of `-e ".[dev]"` succeeds; import guard exercises `ascent.core` |
| pygame headless config removed | `conftest.py` and `test_headless.py` deleted; suite still collects and runs |
| `python3 -m ascent --version` still works | `test_cli.py` (already green) |
| ruff, mypy, format gates | GREEN gate: each exits 0 |

**RED evidence**

```
$ .venv/bin/python -m pytest
FAILED tests/core/test_layering.py::test_importing_core_does_not_load_textual
  AssertionError: ModuleNotFoundError: No module named 'ascent.core'
  assert 1 == 0
1 failed, 2 passed in 0.07s
```

The failure is the guard rejecting a missing `ascent.core` package, not a
collection error, which confirms the harness runs. The two passing tests are the
existing CLI test and the pygame smoke test that this issue removes.

**Anticipated implementation order (GREEN)**

1. `pyproject.toml`: replace `pygame-ce>=2.5` with `textual>=0.60`; drop the
   `pygame.*` mypy override.
2. Create `src/ascent/core/__init__.py` (empty package) so the guard has a target.
3. Delete `tests/test_headless.py` and the pygame session fixture in
   `tests/conftest.py` (remove the file if nothing else needs it).
4. Reinstall into a clean `.venv` and record the gates green: `pytest`,
   `ruff check .`, `ruff format --check .`, `mypy`, and `python3 -m ascent
   --version`.

## Cycle 2: GREEN

**Implementation**

- `pyproject.toml`: the runtime dependency is `textual>=0.60` in place of
  `pygame-ce>=2.5`, and the description reflects the terminal UI. The `pygame.*`
  mypy override is removed; Textual ships type information, so no override is
  needed and mypy stays green with no ignores.
- `src/ascent/core/__init__.py` makes `ascent.core` a real package. It is empty
  of logic (later issues fill it) but carries the layering rule as a docstring so
  the constraint is stated where the code lives.
- `tests/test_headless.py` and `tests/conftest.py` are deleted. Both existed only
  to run pygame windowless under the SDL dummy drivers. Textual UI tests run
  headless through `App.run_test()` and need no shared fixture, so nothing
  replaces them.
- `src/ascent/__init__.py` docstring updated to name Textual.

The `.venv` was rebuilt from `pyproject.toml` (pygame-ce uninstalled, package
reinstalled with `-e ".[dev]"`) so the gates below reflect the declared
dependencies, not leftover state.
