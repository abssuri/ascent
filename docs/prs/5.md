# PR: Issue 5 - Content, map, run, and save

Branch: `5-content-map-run-and-save`. Source: `docs/plans/ascent.md`.

## Summary

Assemble the run around combat: the card content set and starting deck, a seeded
map, the `Run` aggregate that tracks progress and gold, and versioned save/load.
After this issue the whole non-visual game exists as a headless core; only the
Textual shell (issue 6) remains.

The map is a pure function of the run seed (built from its own `RngGroup(seed)`
so it never consumes the live run streams), which means it regenerates
identically on load and needs no serialising. It is a short linear path whose
first node is a combat and whose last is a boss. The `Run` holds the seed, the
run RNG, the map, the player, gold, the deck, and the current node index;
clearing a battle node awards gold from the seeded `card_reward` stream and
advances. Save serialises the run to a JSON-safe dict tagged with `save_version`;
`from_dict` refuses an unknown version with `SaveVersionError` and rebuilds the
map from the seed rather than trusting stored geometry.

## How it was tested

Six tests: two for the map (deterministic regeneration, ends at a boss), one for
run progression (clearing a combat node advances and awards gold), two for save
(JSON round-trip identity, unknown-version refusal), and a guard that every
starting-deck card is defined.

## Cycle 2: GREEN

**Implementation**

- `content.py`: five frozen `Card`s in a name-keyed `CARDS` library, a
  ten-card `STARTING_DECK` of names, and a `card(name)` lookup.
- `map.py`: `NodeType` (`COMBAT`, `ELITE`, `REST`, `BOSS`), the frozen `Node`,
  and `generate_map(seed)`. Generation runs off a fresh `RngGroup(seed)` so it
  stays a pure function of the seed; node 0 is always a combat and the last node
  is always the boss.
- `run.py`: `Run` holding seed, run RNG, map, player, deck, gold, and node index.
  `from_seed` builds a fresh run; `clear_node` awards gold from the `card_reward`
  stream for battle nodes and advances.
- `save.py`: `to_dict`/`from_dict` around `SAVE_VERSION`. `from_dict` checks the
  version first and raises `SaveVersionError` on a mismatch, regenerates the map
  from the seed, rebuilds the deck from names, and coerces the JSON-listified RNG
  stream state back into the tuples `random.setstate` requires.
